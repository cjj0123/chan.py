## 技术实现细节

### 性能监控架构
1. **数据收集层**: 
   - 扫描性能：记录股票数量和耗时
   - 信号质量：记录每个信号的评分
   - 交易执行：记录成功/失败的交易
   
2. **数据处理层**:
   - 实时指标：计算最近30秒的平均性能
   - 历史数据：查询数据库获取历史订单和信号
   
3. **数据展示层**:
   - GUI接口：为前端提供友好的数据格式
   - 报告生成：为邮件报告提供数据支持

### 指标计算算法
1. **P&L计算**: 
   - 买入订单：负现金流（-价格×数量）
   - 卖出订单：正现金流（+价格×数量）
   - 总盈亏：所有现金流的总和

2. **胜率计算**:
   - 按股票代码分组
   - 时间顺序配对买卖订单
   - 盈利交易数 / 总交易数

3. **夏普比率**:
   - 基于日盈亏计算日收益率
   - 年化夏普比率 = 日均收益率 / 日收益率标准差 × √252

4. **最大回撤**:
   - 基于累计盈亏计算净值曲线
   - 最大回撤 = (净值 - 运行最高净值) / 运行最高净值

### HTML报告特性
- **响应式设计**: 适配桌面和移动设备
- **颜色编码**: 
  - 绿色：正面指标（盈利、高胜率、高夏普比率）
  - 红色：负面指标（亏损、低胜率、高回撤）
  - 黄色：警告指标（中等夏普比率）
- **交互友好**: 清晰的表格和卡片布局
- **专业金融风格**: 符合金融报告的专业标准

## 性能影响
- **内存占用**: 轻量级，仅在需要时加载
- **CPU开销**: 后台线程每秒更新一次，开销微乎其微
- **执行效率**: 对主业务逻辑无显著影响

## 使用示例
```python
from Monitoring.PerformanceMonitor import get_performance_monitor

# 获取性能监控器实例
monitor = get_performance_monitor()

# 记录扫描性能
monitor.record_scan_performance(100, 10.0)  # 100只股票，10秒

# 记录信号评分
monitor.record_signal(85.5)

# 记录交易执行
monitor.record_execution(True)

# 获取实时指标
metrics = monitor.get_realtime_metrics()
print(f"扫描速度: {metrics['scan_speed']} 只/秒")

# 获取历史数据
historical_data = monitor.get_historical_performance(hours=24)
```
