# 港股自动化交易系统使用指南

本指南详细说明了如何在现有的 `ashare_bsp_scanner_gui.py` 程序中，使用新集成的港股自动化交易功能。

## 1. 系统架构概述

本系统通过以下三个核心模块协同工作：

1.  **GUI (`App/ashare_bsp_scanner_gui.py`)**: 提供用户交互界面，包含“自动化交易”面板。
2.  **交易控制器 (`App/HKTradingController.py`)**: 封装了完整的“扫描-评分-交易”逻辑，作为独立的后端服务。
3.  **外部服务**:
    *   **视觉 API (`visual_judge.py`)**: 对生成的缠论图表进行智能评分。
    *   **Futu OpenD**: 提供实时行情和交易执行接口。

## 2. 前置条件

在使用前，请确保以下条件已满足：

*   **Futu OpenD 已启动**: 确保 Futu OpenD 客户端正在运行，并且已正确配置了 IP 地址（通常为 `127.0.0.1`）和端口（通常为 `11111`）。
*   **富途自选股已设置**: 在 Futu OpenD 中创建一个名为“港股”的自选股分组（或在 `config.py` 中修改 `hk_watchlist_group` 为您的分组名），并添加您希望监控的港股股票。
*   **环境变量已配置**: 确保 `.env` 文件中包含了正确的视觉 API 密钥等必要信息。
*   **依赖库已安装**: 确保项目所需的所有 Python 依赖库（如 `futu-api`, `pyqt6`, `matplotlib` 等）都已正确安装。

## 3. 操作步骤

### 3.1 启动 GUI 程序

在终端中运行以下命令启动程序：
```bash
python3 App/ashare_bsp_scanner_gui.py
```

### 3.2 使用自动化交易功能

1.  **定位面板**: 在 GUI 的左侧面板底部，找到名为 **“自动化交易 (港股)”** 的新面板。
2.  **启动交易**: 点击 **“启动自动交易”** 按钮。
    *   程序会初始化 `HKTradingController`。
    *   控制器将从富途获取“港股”自选股列表。
    *   对列表中的每只股票进行缠论分析，生成买卖点信号。
    *   为每个信号生成技术分析图表。
    *   调用本地的视觉 API 对图表进行评分。
    *   对于评分达到预设阈值（默认为70分）的信号，程序将自动向 Futu 提交交易订单。
3.  **监控日志**: 所有操作的详细日志（包括分析进度、评分结果、交易指令等）都会实时显示在“自动化交易”面板的日志窗口中。
4.  **停止交易**: 在任何时候，您都可以点击 **“停止自动交易”** 按钮。程序会安全地停止当前的扫描流程，但不会取消已经提交到 Futu 的订单。
5.  **查看结果**: 交易完成后，状态栏会显示本次执行的摘要（如成功买入/卖出的笔数）。

### 3.3 配置参数

关键参数可以在 `config.py` 文件中进行调整：

*   `min_visual_score`: 视觉评分的阈值，只有高于此分数的信号才会被交易。默认值为 `70`。
*   `hk_watchlist_group`: 富途自选股分组的名称。默认值为 `'港股'`。
*   `dry_run`: 设置为 `True` 时，系统只进行分析和评分，但不会实际下单（模拟模式）。默认值为 `False`。

## 4. 注意事项

*   **风险控制**: 自动化交易存在风险。请务必在实盘使用前，先在模拟盘 (`dry_run=True`) 中充分测试。
*   **网络稳定性**: 该系统依赖于与 Futu OpenD 和视觉 API 的稳定网络连接。网络中断可能导致交易失败或延迟。
*   **API 调用限制**: 视觉 API 可能有调用频率限制，请留意日志中的相关提示。
*   **日志文件**: 除了 GUI 中的日志，所有操作也会被记录在 `futu_hk_trading.log` 文件中，方便事后排查问题。

## 5. 故障排除

*   **无法获取自选股**: 检查 Futu OpenD 是否运行，以及自选股分组名称是否正确。
*   **视觉评分失败**: 检查 `.env` 文件中的 API Key 是否有效，以及网络连接是否正常。
*   **交易未执行**: 检查 `dry_run` 参数是否为 `False`，以及账户是否有足够的资金或持仓。
