## 🔧 技术修复

### 问题 1: 头寸计算错误
**现象**: 回测检测到信号但执行 0 笔交易

**原因**: 对于高价股（如腾讯 274 HKD），使用默认每手 100 股和 20% 投资比例时：
- `max_amount = 100000 * 0.2 = 20000 HKD`
- `quantity = int(20000 / 274 / 100) * 100 = int(0.729) * 100 = 0`

**修复**: 修改 `calculate_position_size` 方法，允许买入至少 1 手股票：
```python
def calculate_position_size(self, code: str, current_price: float, available_funds: float, max_investment_ratio: float = 0.2) -> int:
    lot_size = self.lot_size_map.get(code, 100)
    max_amount = available_funds * max_investment_ratio
    
    # 计算可以买入的股数（考虑每手限制）
    affordable_lots = int(max_amount / (current_price * lot_size))
    
    # 如果买不起 1 手，尝试买入最小单位（回测允许不足 1 手）
    if affordable_lots == 0:
        quantity = lot_size  # 至少买入 1 手
    else:
        quantity = affordable_lots * lot_size
    
    return max(0, quantity)
```

### 问题 2: Chan 库内部状态污染
**现象**: `CChanException: kline time err, cur=2024/06/03 10:00, last=2024/06/03 10:00`

**原因**: 每次调用 `trigger_load` 时，Chan 库保留内部状态 (`klu_last_t`)，导致时间比较错误

**修复**: 在每次调用 `trigger_load` 前重置内部状态：
```python
chan_30m.klu_last_t = [CTime(1980, 1, 1, 0, 0) for _ in chan_30m.lv_list]
chan_30m.klu_cache = [None for _ in chan_30m.lv_list]
```

### 问题 3: 调试日志输出过多导致 BrokenPipeError
**现象**: 对比测试中无视觉判断时信号数为 0

**原因**: KLine_List.py 中大量 DEBUG 日志输出导致管道破裂

**修复**: 注释掉 KLine_List.py 中的调试日志输出

## ⚠️ 注意事项

1. **视觉评分简化**: 当前版本只生成 30M 单级别图表，视觉评分固定返回 100 分。如需启用真正的视觉评分，需要恢复 30M+5M 双图表生成。

3. **回测精度**: 回测使用 30 分钟 K 线，可能无法捕捉更细粒度的价格变化。
