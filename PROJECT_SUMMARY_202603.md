🖥️ 一、 交互外壳：缠论交易助手 V2 (TraderGUI)

1. 核心模块与图视面板
📈 扫描器主控 (Scanner Tab):
- 本地 SQLite 节点：支持离线全量/自选股拉取下载，自定义日期回退周期。
- 并发扫描器：选择 30M/5M 等多频率触发，高内聚展示 信号类型 及 视觉/算力评分。

📊 深度分析主控 (Chart Tab):
- 单股走势透视：支持键入任意代码（如 SH.600000 或 US.TSLA），穿透式生成包含 笔、线段、中枢 与 买卖点 的富媒体缠论原生图表。

⚙️ 策略与自动化主控 (Settings Tab):
- 笔严格模式 (Strict Mode)：防线段变形。
- 全流数据源：支持 富途 (Futu)、Baostock 等互补。

2. 生态底层架构 (Architecture)
- 线程安全屏障：全量耗时计算（特别是 Matplotlib 渲染、富途 K线拉取、视觉大模型 I/O）被严格托管在 UpdateDatabaseThread、OfflineScanThread 线程外周。
- 心跳异步槽（Signals）：通过 PyQt6 信号槽实现全息日志分红与安全降级。

--------------------------------------------------------------------------------

⚙️ 二、 全球多核扫描控制器 (Core Controllers)

系统采用模块化分流控制器管控：HKTradingController.py (港股), MonitorController.py (A股), BaseUSTradingController.py (美股)。

1. 并发扫描极速化方案 (asyncio.gather)
- 多路限流阀：使用 asyncio.Semaphore 控制请求爆破点，多线程线程池并发跑视觉评分。
- 前置阈值调优（重点）：在宽屏扫描 (主周期 30M) 产生有效买卖点信号、且时间戳在允许操作窗口（如 3-4 小时之内）时，系统才会**动态增配次级别（如 5M）**的深潜运算，使整体扫描降本增效 3-5 倍。

2. 多源行情协同补救 (Cross-API Fallbacking)
美股模块搭载 多源链路熔断重试。如遇到富途无实时订阅，控制器自动跨接至 Schwab Web-API 免密通道动态反向推算 ATR，确保止损因子在 60 秒内百分之百挂载成功。

🛡️ 三、 防漏风控与动态止损哨兵算法 (Risk Management)

1. 波动率缓冲边界保护 (Floor Clamp)
- 算法漏洞：微幅织布机震荡容易得出极低 ATR（例如 $0.003），任何点差震动都会造成即买即崩。
- 生产环境校正：固定公式加入绝对容错垫底：atr = max(atr, current_price * 0.015)。
- 防护成效：强制抵御 1.5% 空间的纯买卖点差摩擦，彻底终结了“秒抛”事故。

2. 信号延续冷却阻尼器 (Stop-Loss Cooldown)
- 算法漏洞：跌破位或止损斩仓后，下一根折旧蜡烛会把买点时间戳拖至末尾，触发虚假“新信号”再次买入产生反复绞损（拉锯效应）。
- 生产环境校正：当 _check_trailing_stops() 哨兵平仓离场后，系统赋予该 stock 字典标记 self.stop_loss_cooldown[code] = time.time()。
- 防护成效：锁定 10800秒（3小时）绝对物理冷却。彻底剔除在一次溃败信号下，一秒开平仓绞肉机的损耗。

☁️ 四、 跨渠道持久化与第三方熔断共识

- 富途 OpenAPI 隔离区共识：富途 OpenAPI 对美股模拟账户执行 隔离分流控制。下在隔离舱的所有自动化模拟单、实时仓位是 绝对无法 穿透渲染至富途 App/桌面端看盘软件上的。系统已配有完整的 check_sim_account.py 用以离线直观对账。
- API 频次熔断控制：核心 K线读取在 FutuAPI.py / APICached.py 主机循环上，强制在 Loop 迭代头加入了 time.sleep(0.55) 型延迟防守。平滑吸收 60次/30s 的封格指标。
