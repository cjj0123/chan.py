# 缠论量化机器人 - 架构分工与研发准则 (2026)

本项目采用基于智能体角色（Agent Roles）的模块化分工架构，确保策略设计、代码实现与质量校对的高度解耦。以下为核心角色分工说明：

## 🎭 核心角色定义

### 1. 策略架构师 (chanlun_architect)
*   **职责**：负责项目缠论技术逻辑分析、全流选股策略设计以及多周期共振（30M+5M）架构规划。
*   **输出**：策略伪代码、中枢级别对齐方式、风控阈值设定。
*   **核心准则**：严格遵循“物理存档”原则（PLAN.md/STATUS.md）。

### 2. 量化程序员 (chanlun_coder)
*   **职责**：负责 API 对接（Futu, IBKR, Schwab）、下订单指令、指标核算落地。
*   **输出**：高质量 Python 代码、API 频次管理逻辑、数据库持久化层。
*   **核心准则**： Pandas 向量化计算，严禁低效 for 循环。

### 3. 回测与校对员 (chanlun_qa)
*   **职责**：负责核验形态学漏洞、编写回归测试、Alpha 指标测算。
*   **输出**：Pytest 单元测试案例、回测报告分析。
*   **核心准则**：严防线段变形，对标实盘与理论收益偏差。

## 🛠️ 全局研发底线 (Development Bottom Line)

### 🚩 Git 提交准则 (Essential)
> [!IMPORTANT]
> **每次修改代码前，必须显式提交或 Stash 当前工作区的变更 (`git commit`)。**
> 严禁在脏工作区（Dirty Workspace）进行大规模修改或跨模块重构。这不仅是为了版本回溯的安全，更是为了保持审计链路（Walkthrough/Artifacts）的绝对清晰。

### 🚩 语言与沟通准则 (Language Policy)
- **强制要求**：所有智能体在回答问题、生成报告及进行技术同步时，**必须使用中文**。除非是代码中的变量名或必要的英文术语，否则沟通界面应保持 100% 中文化。

### 🚩 交付前质量校验
*   任何代码修改在向主分支合并或向用户汇报前，**必须**运行 `python3 -m py_compile <file>` 进行静态语法通关测试。
*   杜绝因 IndentationError 或 SyntaxError 导致的系统重启失败。

### 🚩 物理进度同步
*   所有智能体架构下的操作必须实时同步至 `PLAN.md`（阶段目标）和 `STATUS.md`（当前状态）。
*   不依赖短期对话上下文，确保开发进度在机器/会话重启后 100% 可恢复。
